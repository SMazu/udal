2. Run-Length Encoding

RLE replaces consecutive repeated values with value-count pairs.

07 / 25

Original region segment in one rowgroup

APAC
350,000 identical values
EMEA
350,000 identical values
AMER
348,576 identical values
After RLE: [(APAC, 350000) | (EMEA, 350000) | (AMER, 348576)]

How it works

Scan the segment top to bottom. Whenever the same value appears in consecutive positions, store one value plus a count instead of repeating the value.

Storage impact

Before: 1,048,576 x 4 bytes = about 4 MB.
After: 3 x value-count pairs = about 24 bytes.
Compression ratio: about 170,000:1 for this column.
RLE Effectiveness And Sorting

RLE works best when physical order creates long runs.

Ordered columnstore indexes reduce segment overlap by sorting order-key values before compression, improving both RLE opportunities and segment elimination.

BEST

Sorted columns with long consecutive runs, such as APAC... EMEA... AMER...

GOOD

Naturally clustered batches, such as one trade date appearing 50,000 times.

PRO TIP

Use Clustered Columnstore Indexes to enforce physical order on your most important filter columns.

POOR

Random high-cardinality columns, such as unique trade IDs or event UUIDs.

3. Dictionary Encoding

Repeated strings become compact integer codes plus one lookup table.

Step 1: build dictionary once per segment

CodeValueOriginal size

Step 2: replace strings with codes

01020312...

Why it compresses

Four distinct values need only 2 bits per row. The strings stay in a small dictionary; the segment stores a dense stream of integers.

Storage impact

Before: 1,048,576 rows x avg 7 bytes = about 7.3 MB.
After: about 40 bytes dictionary + 256 KB codes.
Compression ratio: about 29:1 before other encodings.
Dictionary Encoding Benefits

Storage savings also become CPU and cache advantages.

10 / 25
1Cache efficiency: integer codes fit more rows into L1/L2 cache.
2Faster comparisons: WHERE Region = 'Singapore' becomes code lookup plus integer scan.
3Composable compression: codes can be RLE compressed and bit-packed.
codes: [0, 0, 0, 0, 1, 1, 2] RLE: [(0, 4), (1, 2), (2, 1)] bits: values 0-3 need only 2 bits each

SQL Server specifics

Column segments can use local dictionaries within a rowgroup segment.
Segment metadata exposes encoding type, dictionary IDs, row counts, and min/max statistics.
String columns and repeated values benefit most when segments are well ordered.

In the access layer, prefer constrained categorical columns and stable domain values over free-form strings in high-volume analytical tables.

6. End-to-End Pipeline

Compression moves from raw rows to compact segments plus metadata.

Implementation Playbook

Make query shape and file layout first-class parts of the library contract.

25 / 25

Transformation API rules

1Expose reusable Ibis expressions for metrics, dimensions, and filters.
2Push predicates before joins and aggregations whenever semantics allow.
3Select columns explicitly so engines can perform column elimination.
4Keep transformations lazy until the boundary that materializes results.

Columnar storage rules

Partition by coarse pruning keys such as date, tenant, or source system.
Sort by common filters to improve RLE and min/max segment skipping.
Preserve low-cardinality domains for dictionary encoding and bit-packing.
Use rowgroup-sized writes where the platform supports compressed segments.

The unified access layer is not just portability. It is a place to encode the habits that let columnar engines avoid work.
