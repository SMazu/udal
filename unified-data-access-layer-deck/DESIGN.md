## Style Prompt

Clean technical editorial deck for a data platform architecture review. The visual language should feel precise, executive-ready, and implementation-aware: off-white analytical canvas, ink typography, thin grid rules, compact data diagrams, and a measured use of teal, rust, blue, sage, and gold accents. Slides should read like a senior engineering design walkthrough, not a marketing page.

## Colors

- Canvas: #F6F8F4
- Ink: #1E2A32
- Muted ink: #637077
- Teal accent: #0E6D64
- Rust accent: #D65F3C
- Data blue: #6D9CB5
- Sage fill: #D9E5D6
- Gold highlight: #F1C766
- Panel: #FFFFFF
- Hairline: #D6DED7

## Typography

- Primary: Inter, Avenir Next, SF Pro Display, Helvetica Neue, Arial, sans-serif
- Code and technical labels: IBM Plex Mono, SFMono-Regular, Menlo, Consolas, monospace
- Headlines use strong weight with restrained sizing. Body copy is compact and readable.

## Motion

- Short, calm entrances with small y-offsets and opacity fades.
- Diagrams reveal in operational order: source, expression, compiler, execution, storage.
- Ambient background motion is subtle and never competes with reading.

## What NOT to Do

- No neon-on-dark dashboards or purple-blue gradients.
- No oversized marketing hero sections.
- No decorative card piles or nested cards.
- No low-contrast gray text.
- No visual elements that imply runtime behavior not described by the content.
